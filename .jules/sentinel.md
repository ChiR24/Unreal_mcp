## 2025-03-01 - Missing Path Resolution Verification in C++ Unreal Plugin
**Vulnerability:** In `McpAutomationBridge_EnvironmentHandlers.cpp`, path traversal and path escape vulnerabilities were present because `AbsolutePath` constructed via `FPaths::ProjectDir() / SafePath` was never explicitly resolved with `FPaths::ConvertRelativePathToFull` and strictly checked against the fully resolved project directory path.
**Learning:** `SanitizeProjectFilePath` provides only basic character/traversal sanitization. The final resolved path MUST always be checked to verify it remains within expected project boundaries. A leading slash in `SafePath` can sometimes trick the `operator/` and `ConvertRelativePathToFull` logic if not explicitly bounded.
**Prevention:** Always follow the full pattern for path concatenation: combine, make full absolute path, normalize filename, get normalized project directory, append trailing slash to directory, and assert `AbsolutePath.StartsWith(NormalizedProjectDir, ESearchCase::IgnoreCase)`.
